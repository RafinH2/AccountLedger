# Архитектурные заметки

## Поток транзакции

1. API получает запрос и отправляет команду/запрос MediatR.
2. Обработчик уровня приложения делегирует выполнение в `ILedgerService`.
3. Инфраструктурный слой выполняет транзакционно-безопасную логику в PostgreSQL.
4. Доменные сущности обеспечивают ключевые инварианты (положительная сумма, недостаточность средств, переходы статусов).

## Стратегия согласованности

- Каждая операция, изменяющая деньги, выполняется внутри транзакции БД.
- Строка счёта блокируется через `SELECT ... FOR UPDATE`.
- Идентификаторы транзакций глобально уникальны и используются как ключи идемпотентности.

## Стратегия отката

- Откат создаёт компенсирующую транзакцию (`Type=Revert`) и никогда не удаляет историю.
- Исходная транзакция помечается как `Reverted`.
- Откат зачисления может завершиться ошибкой при недостаточном балансе (чтобы избежать отрицательного баланса).
